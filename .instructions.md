# Padrão Visual — NAPI Abelhas Frontend

Documento de referência para manter a consistência visual em todo o projeto.

---

## Stack de Estilização

- **Tailwind CSS v4** — utility-first, configurado via `@import "tailwindcss"`
- **DaisyUI v5** — component library sobre Tailwind, carregado via `@plugin "daisyui" { themes: all; }` em `globals.css`
- **Temas**: 35+ temas DaisyUI habilitados. O tema ativo é controlado pelo atributo `data-theme` no `<html>` e persistido em `localStorage` (chave `napi-abelhas-theme`).

---

## Atomic Design

O projeto segue o padrão **Atomic Design** com a hierarquia:

| Camada       | Diretório                      | Exemplos                        |
| ------------ | ------------------------------ | ------------------------------- |
| **Atoms**    | `components/atoms/`            | Button, Input, Select, Badge    |
| **Molecules**| `components/molecules/`        | SearchBar, ThemeSwitcher        |
| **Organisms**| `components/organisms/`        | Sidebar, AdminCrudTable, AmostraTable |
| **Templates**| `components/templates/`        | DashboardTemplate               |
| **Pages**    | `app/` (Next.js App Router)    | Cada `page.tsx`                 |

### Regras

- Atoms **nunca** importam de molecules/organisms.
- Molecules são compostas apenas de atoms.
- Organisms podem importar atoms e molecules.
- Templates definem layout (Sidebar + conteúdo), sem lógica de dados.
- Pages conectam dados (hooks/services) com organisms.

## Cores e Variáveis

| Uso                  | Variável/Classe                     |

| Fundo secundário     | `bg-base-200`, `bg-base-200/50`     |

## Componentes — Padrões Visuais

### Botões (`Button` atom)

- Variantes: `primary` (padrão), `secondary`, `accent`, `ghost`, `error`
- Tamanhos: `xs`, `sm`, `md` (padrão), `lg`
- Ícones inline: usar SVG com `className="size-4"` antes do texto
- Ações de tabela: `btn-ghost btn-xs btn-square` com tooltip (`data-tip`)
- Loading: prop `loading` exibe `loading-spinner` automaticamente

### Inputs e Selects

- Sempre com `<fieldset className="fieldset">` + `<legend className="fieldset-legend">`
- Erros: classe `input-error`/`select-error` + `<p className="fieldset-label text-error">`
- Select: sempre `w-full` no `<select>`

### Tabelas

```
card bg-base-100 shadow-sm overflow-hidden
  └─ table w-full
       └─ thead > tr.bg-base-200/60
       └─ tbody > tr.hover:bg-base-200/30.transition-colors
```

- **Não usar** `table-zebra` em tabelas dentro de cards (o hover já dá feedback).
- Cabeçalhos sempre `text-xs uppercase tracking-wider`.
- Coluna de ações: `w-24`, botões `btn-ghost btn-xs btn-square` com SVG icons.
- Paginação: barra inferior com `border-t border-base-200 px-4 py-3`, texto "Página X de Y" à esquerda e botões `join` à direita.

### Cards

- Card padrão: `card bg-base-100 shadow-sm`
- Card de filtro/barra: `card bg-base-200/50 p-4`
- Card clickável: adicionar `hover:shadow-lg transition-shadow cursor-pointer`
- Usar `card-body` para conteúdo interno com padding automático.

### Modais

- Sempre `<dialog className="modal modal-open">` (DaisyUI).
- `modal-box max-w-lg` para formulários, `max-w-sm` para confirmações.
- Botão fechar: `btn btn-sm btn-circle btn-ghost absolute right-3 top-3` com "✕".
- Título: `font-bold text-lg`.
- Subtítulo: `text-sm text-base-content/50 mt-1 mb-5`.
- Footer: `modal-action` com Cancelar (`ghost`) e ação primária.
- Modal de exclusão: ícone centralizado em círculo `bg-error/10` + texto de aviso.

### Alertas de Erro

```html
<div role="alert" className="alert alert-error shadow-sm">
  <svg ...icon... />
  <span>{mensagem}</span>
  <button className="btn btn-ghost btn-xs" onClick={dismiss}>✕</button>
</div>
```

- Sempre com `role="alert"` para acessibilidade.
- Incluir ícone de erro e botão de fechar.

### Estados Vazios

- Ícone grande em círculo: `size-16 rounded-full bg-base-200` com SVG `size-8 text-base-content/30`.
- Título: `font-medium text-base-content/50`.
- Descrição: `text-sm text-base-content/30`.
- CTA opcional: botão `ghost` para criar primeiro registro.

### Loading

- Spinner centralizado: `loading loading-spinner loading-lg text-primary`.
- Texto abaixo: `text-sm text-base-content/40`.
- Padding generoso: `py-16` para dar respiro.

## Badges

- Usar o atom `Badge` com variants semânticos:
  - Status positivo → `success`
  - Status negativo → `error`
  - Informacional → `info`
  - Aviso → `warning`
  - Inativo/neutro → `ghost`
- Para roles/tags: `badge badge-sm badge-outline badge-primary`

---

## Layout

- **Sidebar**: `w-64 bg-base-200 min-h-screen`, fixed na esquerda.
- **Main content**: `flex-1 p-8 bg-base-100`.
- **Espaçamento entre seções**: `space-y-6` (padrão para páginas).
- **Grid de cards**: `grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-4`.

---

## Ícones

- **Usar SVGs inline** (Heroicons v1 mini/solid estilo 20x20 ou outline 24x24).
- Não usar emojis em botões de ação (reservar emojis para títulos de cards decorativos).
- Tamanhos: `size-4` para botões pequenos, `size-5` para alertas, `size-8` para empty states.
- Botões de ação em tabelas: sempre `btn-square` com tooltip.

---

## Responsividade

- Mobile-first: começar com layout de coluna única.
- Breakpoints: `sm:` (flexbox horizontal), `md:` (grid 2 colunas), `lg:` (grid 3 colunas).
- Tabelas: envolver em `overflow-x-auto`.
- Headers de página: `flex-col sm:flex-row` para empilhar em mobile.

---

## Formulários (Clerk)

O `globals.css` contém overrides dos componentes Clerk (`.cl-*`) para respeitar as variáveis do DaisyUI. **Não usar classes Clerk diretamente** — tudo é mapeado automaticamente.

---

## Checklist Antes de Criar Componente

1. Existe um atom/molecule que resolve isso? → Reutilize.
2. As cores usam variáveis DaisyUI? → Nunca hardcode.
3. O loading e empty states estão tratados? → Sempre.
4. O componente funciona em todos os temas? → Teste com light + dark + um colorido.
5. O modal tem botão de fechar e backdrop? → Sempre.
6. As ações de tabela usam ícones SVG (não emojis)? → Sim.
